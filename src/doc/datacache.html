Data Cache

Example | Code | Usage

This widget is useful for caching results for a given type of request. It is capable of specifying intervals and cache invalidation. This can be useful for:

  • data oriented requests that happen throughout the application (especially static/stale datasets)

Reload Data Basic Cache (with keepAlive=30s) Count should increment every ~30s:
count2++; $MQ('l:datacache2.response',{'result':count2})
