Required Options:

type one of the following: text,password,select, dropdown or autocomplete The default is text

Optional Options:

activeClassName The CSS class name to be used when the field is in focus (applies to the entire Field widget)
inactiveClassName The CSS class name to be used when the field is not in focus (applies to the entire Field widget)
fieldActiveClassName The CSS class name to be used when the field is in focus (applies to input element)
fieldInctiveClassName The CSS class name to be used when the field is not in focus (applies to input element)
validator The Validator to be used with the Field widget
message The message that will populate the Field widget value
property The property name within the message that will populate the Field widget value. For "dropdown" or "select" Field widgets this refers to the array property name
text The property name within the message that should be used to populate the option text for a "dropdown" or "select" field
value The property name within the message that should be used to populate the option value for a "dropdown" or "select" field

Field Tags:

header Usage <header>My header HTML</header>
footer Usage <footer>My footer HTML</footer>
error Usage <error>My error HTML</error>

Autocomplete Specific Options:

indicator The element ID of the element where you would like to display indicator that is displayed when the request message is sent and hidden when the response message is received
delay typing inactivity delay before the message is generated (typically a search message). This value should be in milliseconds (e.g. delay="700ms")
autocomplete the message that is triggered once typing stops in the field (e.g., r:my.search.request)
