Search

Example | Code | Usage

The search widget serves as an autocompleter (search as you type).

Here's an example.

$MQ('l:search.response', {result: ['item 1','item 2','item 3']});
You've selected .

Here's another example with a complex search result.

#{name} #{city}, #{job} $MQ('l:search2.response', {result: [{id: 1, name: 'John', city: 'Atlanta', job: 'Engineer'}, {id: 2, name: 'Bob', city: 'New York', job: 'Mayor'}, {id: 3, name: 'Bill', city: 'Seattle', job: 'Plumber'}]});
You've selected an item with id .

Here's the code:

Required Options:

request Request message being sent to search.
response Response message for search results.
selected Message sent when the user has selected an option.

Optional Elements:

key Parameter name used in the request for the query. Defaults to key.
property Property in the response used for the results. Defaults to result
resultId Property to use from the result to send selected message when using complex results. Defaults to id.
inputWidth Width of the input field. Defaults to 200.
delay Delay before firing request message.
indicator Indicator id to show or hide.
activeClass Active class for selecting search results. Defauls to search_result_active.
inactiveClass Inactive class for selecting search results. Defaults to search_result_inactive.
fieldset Fieldset applied to the input element.
name Name applied to the input element.
