Scope is part of the header of every message that is enqueued. This allows for you to restrict messaging beyond the message type. Scope is propigated in both local and remote messages. You can add a scope attribute to elements and the framework will ensure that child elements in the DOM will inherit the scope unless they override it. This attribute will be used for both the "on" handler as well as any messages that are created by the defined on handler or widget.

  • appcelerator: the default scope for all elements if not explicitly annotated
  • *: this means that the on handler will receive messages regardless of scope
