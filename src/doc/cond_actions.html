Client Conditions and Actions

As we stated before, all on attributes must have at least one condition and one action separated by a then like so:

on="[condition] then [action]"

Conditions

Conditions are what trigger the action for a given on attribute definition. A condition is either the receipt of an Appcelerator message or one of the predefined Appcelerator conditions. Below is an example of a Message-based Condition:

Appcelerator also supports a number of predefined conditions based on standard Javascript events. Here's a list:

  • click
  • focus
  • blur
  • dblclick
  • mousedown
  • mouseout
  • mouseover
  • mousemove
  • change
  • keyup
  • keypress
  • keydown
  • contextmenu
  • mousewheel

Actions

Actions define what happens when a Condition is satisfied. Actions can either be Appcelerator Messages or one or more of the supported Appcelerator Actions. Here's a list of Actions that are currently supported:

show Show an element by changing its CSS display property to "block".

Usage: on="[condition] then show"
hide Hide an element by changing its CSS display property to "none".

Usage: on="[condition] then hide"
visible Show an element by changing its CSS visibility property to "visible".

Usage: on="[condition] then visible"
hidden Hide an element by changing its CSS visibility property to "hidden".

Usage: on="[condition] then hidden"
effect Execute a Scriptaculous effect.

Usage: on="[condition] then effect[effect_name]"
toggle Toggle an element's display or visibility property.

Usage: on="[condition] then toggle[display]"
value Set the value of the element. The property that is set will vary based on the element. For example, input fields will have their value attribute set. DIVs will have their innerHTML attribute set. The syntax is value[newValue]

Usage: on="[condition] then value[value_property]"
add Add an attribute or a CSS class to an element.

Usage: on="[condition] then add[class=myclass]"
remove Opposite of add. It will remove an attribute or a CSS class from an element

Usage: on="[condition] then remove[class=myclass]"
set Set the value of any CSS attribute, Javascript object attribute or custom attribute. For example, to set the border of an element, you would do set[border='1px solid #ccc']. To set the selectedIndex of a SELECT element, you would do set[selectedIndex=0]

Usage: on="[condition] then set[attribute=value]"
clear Clear the value for an element

Usage: on="[condition] then clear"
clear Clear the all the elements in a form from an element

Usage: on="[condition] then clearform"
reset Reset the value for an element

Usage: on="[condition] then reset"
popup Display a popup window.

Usage: on="[condition] then popup[id=id_of_popup_elemet]"
enable Enable an element

Usage: on="[condition] then enable"
disable Disable an element

Usage: on="[condition] then disable"
focus Set the focus on the input element

Usage: on="[condition] then focus"
blur Leave the current input element

Usage: on="[condition] then blur"
selectOption Used in SELECT elements. It will set the selected value based on data in the payload

Usage: on="[condition] then selectOption[property]"
javascript run some javascript.

Usage: on="[condition] then javascript[alert('hello')]"
function call a javascript function.

Usage: on="[condition] then function[myfunction()]"
script run some javascript.

Usage: on="[condition] then script[alert('hello')]"
select select the contents of an input field

Usage: on="[condition] then select"
click simulate a button click

Usage: on="[condition] then click[id=my_button_id]"
