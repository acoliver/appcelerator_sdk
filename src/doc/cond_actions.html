Client Conditions and Actions

As we stated before, all on attributes must have at least one condition and one action separated by a then like so:

on="[condition] then [action]"

Actions

Actions define what happens when a Condition is satisfied. Actions can either be Appcelerator Messages or one or more of the supported Appcelerator Actions. Here's a list of Actions that are currently supported:

show Show an element by changing its CSS display property to "block".
hide Hide an element by changing its CSS display property to "none".
visible Show an element by changing its CSS visibility property to "visible".
hidden Hide an element by changing its CSS visibility property to "hidden".
effect Execute a Scriptaculous effect. The syntax is effect[effectName]. For example, effect[Appear]
toggle Toggle an element's display or visibility property.
value Set the value of the element. The property that is set will vary based on the element. For example, input fields will have their value attribute set. DIVs will have their innerHTML attribute set. The syntax is value[newValue]
add Add an attribute or a CSS class to an element. For example, add[class=myclass] or add[myattribute=myvalue]
remove Opposite of add. It will remove an attribute or a CSS class from an element
set Set the value of any CSS attribute, Javascript object attribute or custom attribute. For example, to set the border of an element, you would do set[border='1px solid #ccc']. To set the selectedIndex of a SELECT element, you would do set[selectedIndex=0]
clear Clear the value for an element
reset Reset the value for an element
popup Display a popup window.
enable Enable an element
disable Disable an element
focus Set the focus on the input element
blur Leave the current input element
selectOption Used in SELECT elements. It will set the selected value based on data in the payload - on="r:load.data then selectOption[name]"
